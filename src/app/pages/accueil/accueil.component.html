Abstract

Selon l'Observatoire du respect des obligations d'accessibilité, seulement moins de 5% des sites web sont accessibles pour les personnes handicapées. Dans cette conférence, nous explorons les six erreurs d'accessibilité les plus courantes sur un site web. Afin de les corriger, je vous présenterai des conseils simples à mettre en pratique en HTML et un peu de CSS. Je vous expliquerai aussi comment effectuer un test avec un lecteur d'écran et le clavier afin de vérifier si sa page web est accessible.

Comment commencer à tester ?

Pour trouver les défauts d'accessibilité, on a besoin que de deux choses : le clavier et le lecteur d'écran. L'utilisation de la souris est prohibée.

Pour le lecteur d'écran, on peut utiliser :

  • NVDA (Windows)
  • Orca (EN) (Linux/Ubuntu)
  • VoiceOver (IOS)
  • TallBack (Android)
  • JAWS (Windows)

Pour pouvoir commencer à naviguer sur ce site avec la touche TAB, la solution se trouve dans le fichier style.scss
