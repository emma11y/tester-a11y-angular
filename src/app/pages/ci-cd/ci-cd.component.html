Continuous Integration/Continuous Delivery (CI/CD)

Est-il possible de mettre en place des tests automatisés sur l'accessibilité numérique ?

Oui, c'est possible. Il existe des tests automatisés qu'on peut mettre en place dans les CI/CD.

Attention, les tests automatisés ne couvrent pas tous les critères d'accessibilité. Le seul moyen de s'assurer que le site est accessible passe par les tests utilisateurs.

Les tests automatisés ne peuvent pas vérifier :

  • si c'est la bonne langue,
  • si les textes alternatifs des images sont pertinents ;
  • si les labels des boutons et liens sont pertinents ;
  • si les éléments reçoivent correctement le focus du clavier.

Comment mettre en place un test automatisé avec Github Actions ?

Sur ce site, vous pouvez exécuter Axe et Pa11y qui se basent sur les normes internationales d'accessibilité WCAG.

  • En lancant une ligne de commande en local :
    • axe http://localhost:4200/
    • pa11y http://localhost:4200/
  • ou créer un fichier yml qui analysera votre code avec GitHub Actions

Source : Automating the accessibility tests of your source code with GitHub Actions
